# Headwinds UI Audit — July 17, 2026

Scope: all 45 UI files (27,659 lines) statically analyzed, plus a visual pass of the live game
(lobby, Dashboard, Market, Route Planner, Routes, Fleet, Operations, Rivals, Finance) at 1523px.

**The verdict:** the game does not look AI-generated at the structural level. Routes, Finance,
Fleet, and the lobby read as professional dashboard UI. What creates the "AI slop" impression is
five specific habits, all fixable: over-explaining prose, the em-dash tic, native browser popups,
repeated boilerplate, and small consistency bugs that a human QA pass would have caught. The bugs
matter most: nothing says "nobody looked at this" like "1 countries" on the main screen.

---

## P0 — Visible bugs (each one screams "unreviewed")

1. **"1 countries" on the Dashboard.** `Dashboard.jsx:457` renders `${countries.length} countries`
   with no singular form. Currently visible on screen.

2. **Double wrench in mechanical-failure toasts.** Toast shows "🔧 🔧Mechanical Failure — …".
   `reducer.mjs:2193` embeds `${f.icon}` in the title AND passes `icon: f.icon` separately, so the
   toast renders the icon twice, and the second one has no space after it. Drop the icon from the
   title string.

3. **"Headwinds v0.0.0 · build dev" in the production footer.** The `__APP_VERSION__`/`__BUILD_ID__`
   defines exist only in the root vite config, not in `apps/headwinds-web/vite.config`. Every page
   of the live game footer says it's a dev build.

4. **Duplicate aircraft auto-names.** Fleet list shows two different airframes both named
   "Airbus A320neo #18" (tails NAUS38 and NAUS01). The auto-numbering can collide, so renaming or
   selling by name is ambiguous.

5. **Rivals tab contradicts the lobby.** The world lobby standings list 3 airlines (one abandoned,
   one bankrupt); the in-game Rivals tab says "You're the only airline in this world so far."
   Filtered differently in each place. Either show inactive rivals greyed out or say
   "No active rivals" instead of "only airline".

6. **Two different "idle" counts on one screen.** Dashboard alert says "16 idle aircraft"; the
   Fleet Utilisation card below says "Idle 20". (One counts fully-unassigned aircraft, the other
   counts low-utilization ones.) Pick one number or label them differently.

## P1 — The actual "AI slop" tells

7. **Native browser confirm() dialogs (23 call sites).** Selling aircraft, cancelling orders,
   closing routes, save slots, and every admin action in the lobby all pop the OS-chrome
   `window.confirm` box, several with multi-paragraph `\n\n` text crammed into it. This is the
   single cheapest-feeling interaction in the game, and the codebase already has a styled confirm
   modal (the New Game dialog in App.jsx) to generalize. Sites: Fleet ×7, lobby App ×8, Routes ×2,
   Marketplace, HubManagement, CargoRoutesList, SaveLoadModal, Finance, Messages.

8. **The em-dash tic: 379 UI strings still use " — ".** The market/aircraft pass removed them from
   descriptions, but they're everywhere else: section headers ("PROFIT WATERFALL — REVENUE TO
   WEEKLY CASH", "INDUSTRY LEADERBOARD — MARKET CAPITALISATION"), toasts, alerts, empty states,
   labor-negotiation copy. Worst files: Wiki 73, Finance 51, Routes 25, Operations 22, lobby
   App 20, Dashboard 19, OnboardingTour 18. Convention to adopt: "·" or ":" in headers and labels,
   plain sentences in prose.

9. **Over-explaining prose (1,246 strings over 180 chars).** The pattern: every card gets a
   3–4 line paragraph restating what the numbers already say. Worst offenders, in player-visible
   order:
   - **Operations labor cards**: each has a long description PLUS the identical italic disclaimer
     "Variable flight duty pay … this covers fixed overhead only" repeated verbatim on both Pilots
     and Cabin Crew. Show it once, or make it a tooltip.
   - **Onboarding tour**: most steps are 3+ sentences; players skim tours.
   - **Wiki articles**: fine to be long, but they open with restated intros.
   - Rule of thumb: card descriptions get one line; detail moves to a tooltip or Help.

10. **Raw emoji leaks.** Most emoji route through GLYPH_MAP → SVG icons (good system), but:
    - Operations renders `{group.emoji}` raw in each labor-group header (🧑‍✈️, 🛎️ on screen).
    - Glyph falls back to raw text for 🏗 (HubManagement), 🗑 (CabinTemplatePicker),
      🎁 (AircraftCheckout), ✏ (rename buttons), ＋ (SetupScreen/Branding) because they're not in
      GLYPH_MAP.
    - Airports.jsx uses raw 🌍🌎🌏 for regions.
    Add them to GLYPH_MAP or swap for existing SVG icons.

11. **Dashboard KPI redundancy and ragged grid.** 10 stat tiles wrap into a second row holding just
    2. The DATE tile duplicates the date already pinned in the header 60px above it, and the CASH
    tile duplicates header cash. Cut the DATE tile (move "-$5.95M last wk" into the profit tile),
    consider cutting CASH, and you get a clean single-row-or-2×4 grid. This was loge999's exact
    "redundant info" example.

## P2 — Layout and density

12. **Nav overflow with no affordance.** 16 top-level tabs; at 1523px wide the Help tab is cut off
    and there's no scroll hint. Deepest fix for "fit content at various resolutions": group tabs
    (e.g. Network = Routes/Gates/Hubs, Company = Operations/Reputation/Loyalty) or add a "More ▾"
    overflow menu; cheapest fix: icon-only tabs below ~1600px.

13. **Financial History chart wastes half its card.** The plot area is a fixed ~560px hugging the
    right side; the left half of the full-width card is empty at desktop widths. Make the chart
    stretch to the card.

14. **Market table headers scroll away.** The new table's sticky header only sticks inside its own
    scroll wrapper, so on normal page scroll the column labels disappear (confirmed live). Restructure
    so the header sticks below the nav, or cap the table height with internal scroll.

15. **Rivals tab stacks two full-height empty states** (Contested Routes, Rival Networks) so the
    screen is ~80% empty. Collapse empty sections to single-line hints until they have content.

16. **Finance has 11 sub-tabs** (P&L, Cash Flow, Unit Economics, Forecast, Trends, Statistics,
    Balance Sheet, Loans, Fuel, By Route, By Airport). Consider grouping into 3–4 (Statements,
    Planning, Analysis, Detail) with secondary switches inside.

## What already reads as human (don't touch)

The Routes table, Finance waterfall + P&L, Fleet list view (search, filters, thumbnails), the new
Market table, the merged single header, and the lobby standings all look like someone with taste
built them. The dark palette, uppercase micro-labels, and stat-tile language are consistent across
screens; the polish gap is purely in the items above.

## Suggested order of attack

1. P0 items 1–6: a day of small fixes, disproportionate credibility gain.
2. Item 7 (confirm modals): one shared component, replace 23 call sites.
3. Items 8–9 on the five worst files (Wiki, Finance, Operations, Routes, lobby): copy-only pass.
4. Items 11–12: Dashboard tile cut + nav overflow handling.
5. The rest opportunistically.
